01 install

Install NxAgent

Use the package name intended for the Python release. Until publishing is complete, treat this as the stable install target for examples and docs.

pip install nx-agent
02 concepts

The core model

NxAgent keeps the mental model small: define agents, attach tools, compose a workflow, run it, then inspect what happened.

Agent

An agent represents a focused worker with a role, goal, model, instructions, and optional tools.

tool

A decorated Python function that agents can call during a run. Type hints and docstrings become the callable schema.

Workflow

A coordinated execution path for one or more agents, including ordering, routing, tracing, and final output handling.

Run

A single execution with input, tool calls, model responses, intermediate steps, and final output.

03 agents

Define agents with clear responsibility

Agents should be narrow enough to guide behavior, but broad enough to complete useful work. Start with role and goal, then add tools or model settings only when needed.

from nx_agent import Agent agent = Agent( role="Researcher", goal="Find accurate information and explain it clearly", instructions="Prefer concise answers with source notes." ) result = agent.run("Summarize the current state of browser automation.") print(result)
04 tools

Expose Python functions as tools

Use lowercase @tool for decorator-style tools. NxAgent should infer the tool name, description, argument schema, and return shape from normal Python code.

from nx_agent import Agent, tool @tool def web_search(query: str, limit: int = 5) -> str: """Search the web for current information.""" ... # call your search provider agent = Agent( role="Research Analyst", goal="Find and synthesize reliable information", tools=[web_search], )
Tool design: keep names action-oriented, type every argument, and use docstrings to tell the model when the tool should be called.
05 workflows

Coordinate multiple agents

Workflows are where single agents become production systems. Use them to define collaboration, sequencing, review loops, and routing behavior.

from nx_agent import Agent, Workflow, tool @tool def web_search(query: str) -> str: """Search the web for current information.""" ... # your implementation researcher = Agent( role="Research Analyst", goal="Collect accurate source material", tools=[web_search], ) writer = Agent( role="Technical Writer", goal="Turn findings into a clear brief", ) workflow = Workflow(agents=[researcher, writer]) result = workflow.run("Create a short brief on agent observability.") print(result)
06 configuration

Configure models, memory, and limits

Configuration should stay close to the object it affects. Keep common defaults at the workflow level, then override specific agents when their behavior differs.

agent = Agent( role="Analyst", goal="Summarize findings", model="openai:gpt-4.1", temperature=0.2, max_steps=6, ) workflow = Workflow( agents=[agent], memory="session", tracing=True, )
07 runtime

Run, stream, and observe

Production agent workflows need visibility. The runtime should make final output easy to consume while preserving trace data for debugging.

Run once

result = workflow.run("Draft a release note.") print(result.output) print(result.steps)

Stream events

for event in workflow.stream("Research this topic"): print(event.type, event.data)
08 examples

Common workflow patterns

These examples show the kinds of applications NxAgent is designed to support as the framework grows.

research

Research brief

Researcher gathers source material, reviewer checks gaps, writer produces a concise brief.

assistant

Tool-using assistant

Agent calls APIs, reads files, updates systems, and returns a traceable answer.

review

Review loop

Builder agent drafts an output, reviewer agent critiques it, finalizer agent prepares the response.

routing

Dynamic routing

Workflow routes tasks to specialist agents based on intent, required tools, or confidence.

api preview

Current API surface

The early public surface should stay small and predictable while deeper runtime features are added.

Agent(...) tool(...) Workflow(...) .run(...) .stream(...)
